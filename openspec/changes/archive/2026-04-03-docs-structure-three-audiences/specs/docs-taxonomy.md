# Spec: Documentation Taxonomy

## Три аудитории

### Аудитория 1: Человек-пользователь
- **Директория:** `docs/user/`
- **Кто:** программист, пишущий `.sno` код вручную
- **Что:** туториалы, языковые конструкции с объяснениями, примеры
- **Стиль:** обучающий, с контекстом "почему", на русском и английском
- **Не включает:** внутреннюю документацию компилятора, формальные спецификации

### Аудитория 2: LLM-пользователь
- **Директория:** `docs/llm/`
- **Кто:** языковая модель (Claude, GPT и др.), генерирующая `.sno` код
- **Что:** quick reference cards, таблицы операторов, аксиомы, GBNF, ментальные модели
- **Стиль:** структурированный, без prose, ≤1800 токенов на файл
- **Не включает:** туториалы, историю проекта, научные обоснования

### Аудитория 3: LLM-разработчик (внутренняя)
- **Директория:** `context/`
- **Кто:** Claude agent, строящий компилятор
- **Что:** Rules, Architecture, Project State, Development Guide, Phases
- **Стиль:** технический, командный, структурированный
- **Не включает:** пользовательскую документацию

## Дополнительные директории (без изменений)

| Директория | Аудитория | Назначение |
|------------|-----------|-----------|
| `docs/specs/` | Дизайнеры языка | Формальная спецификация |
| `docs/research/` | Исследователи | Научные обоснования |
| `docs/articles/` | Общество | Образовательная серия (outreach) |
| `docs/testing.md` | Internal QA | Руководство по тестированию |
| `docs/stress-server.md` | Internal QA | HTTP-дэшборд тестов |

## Правило размещения

При создании нового документа:
1. Определи аудиторию: человек-пользователь / LLM-пользователь / LLM-разработчик
2. Помести в соответствующую директорию
3. Не смешивай аудитории в одном файле

## Структура docs/user/ (скелет)

```
docs/user/
  README.md          ← точка входа, оглавление
  quickstart.md      ← первые шаги (TODO)
  syntax.md          ← языковые конструкции (TODO)
  examples.md        ← примеры программ (TODO)
```

Файлы TODO создаются по мере готовности — не сейчас.
