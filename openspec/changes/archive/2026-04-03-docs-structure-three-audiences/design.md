# Design: Documentation Taxonomy

## Ключевые решения

### D1: context/ остаётся как есть
`context/` — уже правильно изолированная директория для LLM-разработчика.
Альтернатива: переименовать в `docs/internal/` — отклонено, сломает CLAUDE.md и привычки.

### D2: docs/user/ — новая директория, только скелет
Создаём `docs/user/README.md` как точку входа.
Туториалы (quickstart, syntax, examples) — TODO, не создаём пустые файлы.
Причина: не плодить файлы без контента (RULES.md, правило 5).

### D3: docs/llm/ не трогаем структурно
`docs/llm/synoema.md` уже существует. Директория правильно названа.
Добавляем только: упоминание в CLAUDE.md и RULES.md.

### D4: Правило в RULES.md — новая секция 7
Добавляем в `context/RULES.md` секцию "7. Документация" с таблицей аудиторий.
Альтернатива: только в CLAUDE.md — отклонено, RULES.md — нормативный документ для правил.

### D5: CLAUDE.md — добавляем секцию и обновляем таблицу навигации
- Новая секция "## Документация" после "## Навигация"
- Таблица трёх аудиторий
- Обновить таблицу навигации: добавить строки для docs/user/ и docs/llm/

### D6: docs/user/README.md — язык
Русский (как основной язык проекта) + английские термины для кода.

### D7: Не переносить testing.md и stress-server.md
Они внутренние, но перемещение создаёт diff без ценности.
Достаточно зафиксировать в таксономии как "Internal QA".
