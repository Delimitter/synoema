# Proposal: Documentation Structure for Three Audiences

## Problem

Документация Synoema смешана в одной директории `docs/` без чёткого разделения по аудитории.
Три принципиально разных читателя получают один и тот же неструктурированный набор файлов:

1. **Человек-пользователь** — программист, пишущий `.sno` код вручную
2. **LLM-пользователь** — языковая модель, *генерирующая* `.sno` код
3. **LLM-разработчик** — Claude agent, *строящий* компилятор Synoema

Текущие проблемы:
- `docs/testing.md` и `docs/stress-server.md` — внутренняя документация, лежит рядом с `docs/articles/` (outreach)
- `docs/llm/` существует, но не задокументирован как отдельная аудитория в CLAUDE.md или RULES.md
- Нет `docs/user/` — у человека-пользователя нет чёткой точки входа
- CLAUDE.md не фиксирует правила "что куда писать" для документации
- `context/` правильно изолирован для LLM-разработчика, но это нигде явно не сказано

## Goal

Зафиксировать трёхуровневую модель документации в правилах и навигационных файлах:

1. Обновить **CLAUDE.md** — добавить секцию "Документация" с таблицей трёх аудиторий
2. Обновить **context/RULES.md** — добавить правило 7 "Документация"
3. Создать **docs/user/README.md** — точка входа для человека-пользователя
4. Обновить навигационную таблицу в CLAUDE.md

## Аудитории и принципы

### Человек-пользователь (`docs/user/`)
Программист, который пишет `.sno` код вручную или читает его.
- Туториалы, языковые конструкции с объяснениями, примеры
- Стиль: обучающий, с контекстом "почему"
- Не дублирует спецификацию — ссылается на неё

### LLM-пользователь (`docs/llm/`)
Языковая модель (Claude, GPT и др.), которая *генерирует* `.sno` код.
- Quick reference cards, таблицы операторов, аксиомы, ментальные модели
- Стиль: максимально структурированный, без prose, ≤1800 токенов
- Цель: override дефолтного поведения LLM (no `if`, no `def`, etc.)

### LLM-разработчик (`context/`)
Claude agent, который *строит* компилятор Synoema.
- Rules, Architecture, Project State, Development Guide, Phases
- Уже правильно изолирован — не трогать структуру

## Non-goals

- Перемещение существующих файлов docs/ (docs/articles/, docs/specs/, docs/research/ — без изменений)
- Создание контента для docs/user/ (только скелет)
- Изменение компилятора или тестов

## Output files

- `CLAUDE.md` — обновлён (секция документации + навигация)
- `context/RULES.md` — обновлён (правило 7)
- `docs/user/README.md` — создан (скелет)
