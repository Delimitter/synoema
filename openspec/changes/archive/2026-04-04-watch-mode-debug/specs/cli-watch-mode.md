# CLI Watch Mode

## Capability: cli-watch-mode

Команда `watch` для автоматического перезапуска подкоманды при изменении .sno файла.

## CLI Interface

```
synoema watch <subcommand> <file> [--interval <ms>] [--clear]
```

- `<subcommand>`: `run`, `jit`, `build`, `test`
- `<file>`: путь к .sno файлу (или директории для `test`)
- `--interval <ms>`: интервал polling (по умолчанию 500ms)
- `--clear`: очищать терминал перед каждым перезапуском (по умолчанию: да)
- `--no-clear`: не очищать терминал

## Behavior

1. Первый запуск: выполнить подкоманду, показать результат
2. Цикл: каждые N ms проверить mtime файла
3. Если mtime изменился — перезапустить подкоманду
4. Для `run`: отслеживать также все imported файлы (через resolve_imports)
5. При ошибке — показать ошибку, продолжить watch (не exit)
6. Ctrl+C — завершение

## Debug Info Header

Перед каждым запуском показывать:
```
[watch #3] 14:32:05 | src/main.sno (modified) | 12ms
────────────────────────────────────────────────
```

- `#N` — номер перезапуска (начиная с 1)
- Время запуска
- Путь файла + (modified) при изменении / (initial) при первом
- Время выполнения предыдущего запуска (после первого)

## Constraints

- Без новых зависимостей (std only polling)
- Не модифицировать существующие команды run/jit/build/test
- Переиспользовать существующие функции run_file/jit_file/build_file
