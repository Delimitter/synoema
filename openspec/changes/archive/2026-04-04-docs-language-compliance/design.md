# Design: Documentation Language Compliance

## Decisions

### D1: Правило 7d — формулировка

Добавить после секции 7c в RULES.md:

```markdown
#### 7d. Язык документации

| Аудитория | Язык | Стиль |
|-----------|------|-------|
| LLM-пользователь (`docs/llm/`) | Английский | Минифицированный: таблицы, списки, сниппеты. Без prose |
| Человек-пользователь (`README.md`, `docs/`, `CONTRIBUTING.md`) | Английский | Human-readable: полные предложения, объяснения |
| LLM-разработчик (`context/`, `CLAUDE.md`) | Русский | Произвольный (prose или минифицированный) |
```

### D2: Перевод docs/testing.md

Полный перевод на английский с сохранением структуры, таблиц, команд. Стиль: human-readable prose (пользовательский документ). Все секции, примеры кода и CLI-команды остаются идентичными.

### D3: Перевод docs/stress-server.md

Полный перевод на английский с сохранением структуры. ASCII-диаграммы, таблицы, код — без изменений. Только текст переводится.

### D4: Минификация docs/llm/synoema.md

Удалить prose-элементы, оставить содержание:
- Заголовок первой секции "Mental model: what NOT to write" → "1. Overrides" (terse)
- Убрать пояснительные фразы типа "Override your Haskell/Python priors first"
- Секция "Core axioms" — оставить bullet-list, убрать дефисные пояснения длиннее одной строки
- Код-блоки — оставить без изменений (уже минимальны)
- Prose-комментарии внутри кодовых блоков — оставить (они part of примера)

**НЕ менять:** структуру секций, порядок, содержание таблиц, примеры кода.

## Риски

- Перевод testing.md может потерять нюансы — митигация: сохраняем терминологию Rust/cargo
- Минификация synoema.md может убрать полезный контекст — митигация: минифицируем только prose-обёртку, не content
