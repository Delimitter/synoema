# Design: MCP Performance Improvements

## D1: Механизм раннего выхода в search

**Решение:** Изменить `collect_rs_files` и `collect_files_by_ext` — коллбэк возвращает `bool` (true=continue, false=stop). Рекурсия прерывается при false.

**Альтернативы:**
- Итератор ленивый (impl Iterator) — сложнее реализовать для рекурсивного обхода, требует Box<dyn Iterator>
- Arc<AtomicBool> как стоп-флаг — overhead атомарных операций, overcomplicated для однопоточного сервера
- Собрать все пути заранее, потом итерировать — не решает проблему, та же O(n_files)

**Обоснование:** Минимальное изменение сигнатуры. Рекурсивный обход уже есть — нужно только добавить возвращаемое значение и проверку.

## D2: Кэш all_crates — отдельный Mutex

**Решение:** Добавить `crates_cache: Mutex<Option<(SystemTime, Vec<CrateSummary>)>>` в `LiveIndex`. Отдельный от файлового кэша Mutex.

**Альтернативы:**
- Один Mutex на всё (файлы + crates_cache) — deadlock risk при вызове all_crates → get_crate → get_file (get_file уже берёт self.cache lock)
- RwLock — overcomplicated для данного паттерна

**Обоснование:** Два Mutex безопасны т.к. упорядочены: `crates_cache` lock берётся и отпускается до `self.cache` lock (через `get_crate`).

## D3: Serde structs для doc_query

**Решение:** Создать `#[derive(Serialize)]` структуры прямо в `dev_tools.rs` (не в отдельном модуле). Приватные, используются только для сериализации.

**Альтернативы:**
- Использовать `serde_json::json!()` макро — менее типобезопасно, сложнее рефакторить
- Внешний модуль `doc_types.rs` — overhead для 5 структур

**Обоснование:** Структуры в том же модуле — минимальный blast radius, serde уже в зависимостях.

## D4: Truncation last_error

**Решение:** Обрезка при записи в `on_tool_result`, не при чтении. Константа `MAX_ERROR_LEN = 500`.

**Обоснование:** Обрезка при записи гарантирует, что хранилище не растёт. При чтении — уже поздно, память уже аллоцирована.

## Риски

| Риск | Митигация |
|------|----------|
| Early exit пропустит лучшие результаты в конце | Приемлемо — search уже возвращает первые 5, порядок файлов недетерминирован |
| crates_cache stale при изменении файлов внутри крейта | Кэш инвалидируется только по mtime директории; файловый кэш handle-ит отдельно |
| serde structs увеличат LOC | ~30 строк struct definitions vs ~20 строк удалённой esc() — delta +10 |
