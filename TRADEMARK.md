# Synoema Trademark Policy

This document outlines the policy for use of the Synoema project trademarks. This policy is modeled after the Rust, Mozilla, and Docker trademark guidelines.

## Marks Covered

The following are trademarks of Andrey Bubnov:

- **"Synoema"** — word mark for the programming language
- **"sno"** — word mark for the command-line interface
- **File extensions:** `.sno` (source), `.snm` (module interface), `.snp` (package) — associated with the project
- **Synoema logo** — when created, all logo variants

## Permitted Uses (No Permission Needed)

You **do not** need written permission to:

- **Refer to Synoema** in articles, blog posts, talks, academic papers, or social media (e.g., "I wrote a parser in Synoema")
- **State compatibility** with Synoema (e.g., "works with Synoema", "supports .sno files", "Synoema syntax highlighter")
- **Use the name in package descriptions** (e.g., "Synoema syntax highlighter for VS Code", "Synoema formatter")
- **Distribute unmodified copies** of official Synoema releases with the original name and branding intact
- **Use in comparative statements** that are factual and non-misleading (e.g., "Synoema vs. Python for LLM code generation")
- **Discuss or critique** the project by name

## Requires Written Permission

You **must** obtain written permission from the trademark holder to:

- Use "Synoema" or "sno" **in a product name or service name** (e.g., "Synoema Cloud", "sno Pro")
- Create a **modified distribution** and call it "Synoema" (see Fork Naming below)
- Use the Synoema logo in **merchandise or commercial products**
- **Imply endorsement** by the Synoema project or its author
- Use the marks in **advertising** in a way that suggests official partnership
- Use the marks as part of a **domain name** for a commercial service

## Never Permitted

The following uses are **never** permitted, regardless of permission:

- Registering domain names containing "synoema" for competing projects or cybersquatting
- Creating **confusingly similar** marks (e.g., "Synoema++", "SynoemaOS" without permission)
- Implying **official affiliation** without explicit written agreement
- Using the marks in a manner that **disparages** the project or its contributors
- Using the marks in connection with **illegal activities**

## Fork Naming Guidelines

If you fork Synoema:

1. You **MUST rename** your fork. Do not distribute a modified version under the name "Synoema" or "sno"
2. You **MAY state** the relationship: "based on Synoema" or "forked from Synoema" in your README and documentation
3. You **MUST NOT** use the Synoema logo or suggest that your fork is the official distribution
4. You **SHOULD** clearly indicate what modifications you have made

**Examples of acceptable fork names:** "MyLang (based on Synoema)", "SnoFork"
**Examples of unacceptable fork names:** "Synoema 2.0", "Synoema Enterprise", "sno-pro"

## Logo Usage Guidelines

When the Synoema logo is available:

- Do not alter the logo colors, proportions, or design elements
- Maintain clear space around the logo (minimum: height of the "S" in the wordmark)
- Do not place the logo on visually busy backgrounds
- Use official logo files from the project repository
